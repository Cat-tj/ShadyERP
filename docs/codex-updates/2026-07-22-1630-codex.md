# 2026-07-22 16:30 Codex

## Fokus

Memperbaiki routing subdomain vertical Altora agar tenant prebuilt tidak jatuh ke flow umum/kasir.

## Root Cause

- `src/app/pilih-aplikasi/page.tsx` masih mengarahkan semua tenant `SIMPLE` ke `/kasir` untuk staff dan `/simple/hari-ini` untuk owner/manager tanpa membaca subdomain.
- `src/components/simple-shell.tsx` memakai tab global `Kasir/Uang/Data/Lainnya` untuk semua tenant, sehingga `teams.altora.my.id` dan `pabrik.altora.my.id` tetap terlihat seperti POS.
- Register masih menyediakan tahap `Kustomisasi` untuk tenant umum, sementara arah produk sekarang adalah vertical prebuilt.
- Server action register masih menerima `disabledModules` dari hidden input. Sekarang preset modul dihitung di server dari vertical/subdomain atau business mode.

## Perubahan

- Menambahkan `src/lib/vertical-navigation.ts` sebagai sumber tunggal default route dan primary action per vertical.
- `pilih-aplikasi` sekarang redirect berdasarkan vertical:
  - Teams: `/hris` atau `/absensi` untuk staff.
  - Pabrik: `/produksi`.
  - Accounting: `/finance`.
  - Laundry: `/laundry`.
  - Jasa: `/booking`.
  - Cafe/Toko/E-commerce/Supermarket/Counter: tetap ke ringkasan/kasir sesuai role.
- `SimpleShell` sekarang menerima `vertical`, memakai logo sub-brand, dan menampilkan tab per vertical.
- `RegisterForm` disederhanakan menjadi 2 tahap. Tidak ada lagi layar kustomisasi fitur.
- `registerAction` menghitung modul aktif secara server-side, bukan dari hidden input.
- Metadata `Altora Pabrik` sekarang menyertakan modul `produksi`.
- Root landing `altora.my.id` tetap bisa dibuka meski user sedang login; redirect otomatis hanya terjadi di subdomain vertical.
