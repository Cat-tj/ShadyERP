# Codex Update - 2026-07-23 10:00 WITA

## Scope

Port the HRIS KPI visibility behavior into the ShadyERP release repository. This is deliberately not a deployment note for `svt-kpi-monitor`.

## Changes

- Added KPI scopes: `INDIVIDUAL`, `DEPARTMENT`, and `ORGANIZATION`.
- Added a migration that keeps existing KPI goals intact as individual goals while allowing organization and department targets without a required worker.
- Added server-side KPI visibility policy in `src/server/services/kpi-goal-service.ts`:
  - Owner sees all tenant KPI goals.
  - Staff sees only their own individual KPI goals.
  - Managers see direct-report KPI goals and their own department goals.
  - Managers assigned to department `CD IMM` also see organization KPI goals.
- Added `/hris/kpi` with create-target and update-progress controls.
- Added `Target & KPI` to the HRIS sidebar and changed the HRIS feature card to point to the usable KPI page.

## Data prerequisites

Manager visibility relies on an active `Worker` whose `Person.email` matches the signed-in User email, plus active `Assignment` rows. Direct reports need `Assignment.managerId` set to that manager Worker ID.
