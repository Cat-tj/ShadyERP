# 2026-07-06 21:04 - Codex Update

## Ringkasan

Memperbaiki UI/UX halaman Kasir yang broken di desktop/tablet.

## Masalah

- Panel invoice muncul terlalu cepat di breakpoint tablet/desktop sedang.
- Grid produk masih memaksa 2 kolom besar sehingga tombol qty produk keluar dari card.
- Tombol `+` produk bisa terlihat seperti melayang di luar kartu.
- Visual produk terlalu besar untuk ruang kartu pada breakpoint tertentu.

## Implementasi

- Mengubah layout utama POS dari `md:flex-row` ke `xl:flex-row`.
  - Invoice kanan hanya tampil di layar lebar.
  - Tablet dan desktop sempit tetap fokus ke grid produk + bottom/cart sheet.
- Mengubah grid produk menjadi:
  - 1 kolom di mobile.
  - 2 kolom di tablet/desktop sedang.
  - 3 kolom di layar sangat lebar.
- Mengubah kartu produk menjadi grid dengan kolom visual dan konten yang stabil.
- Membuat area harga truncate dan kontrol qty fixed width.
- Menyesuaikan ukuran visual produk agar tidak overflow.

## Verifikasi

- `npx tsc --noEmit` berhasil.
- `npm run build` berhasil.
