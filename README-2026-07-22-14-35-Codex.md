# Update 2026-07-22 14:35 - Codex

## Routing Auth Lintas Subdomain

### Masalah

Logout pada domain produksi dapat mengarahkan browser ke `localhost:3000/login`
ketika URL basis Auth.js di VPS masih menggunakan nilai lokal.

### Perubahan

- Semua tombol logout menggunakan origin browser saat ini sebagai callback.
- Auth.js hanya menerima callback `https://altora.my.id` dan seluruh subdomain
  `https://*.altora.my.id`; URL asing atau nilai lokal di produksi ditolak.
- Login dan auto-login sesudah register juga membangun redirect absolut dari host
  request saat ini agar tetap berada pada produk yang sama.
- Resolver URL absolut untuk QR mengikuti `x-forwarded-host` dan
  `x-forwarded-proto` dari Caddy.

### Dampak

- `cafe.altora.my.id` keluar ke `cafe.altora.my.id/login`.
- `toko.altora.my.id`, `laundry.altora.my.id`, `counter.altora.my.id`,
  `pabrik.altora.my.id`, dan `company.altora.my.id` mengikuti pola yang sama.
- `localhost` hanya tetap tersedia ketika benar-benar menjalankan development
  lokal, tidak pernah sebagai fallback produksi.

### Verifikasi

- `npm run build`
- `npx tsc --noEmit`
- `npx eslint src/lib/auth-client.ts src/lib/auth.config.ts src/lib/base-url.ts src/app/login/actions.ts src/app/register/actions.ts src/components/app-shell.tsx src/components/simple-shell.tsx src/components/hub-picker.tsx`

### Catatan VPS

Pastikan `AUTH_URL`, `NEXTAUTH_URL`, `AUTH_URL_INTERNAL`, dan
`NEXTAUTH_URL_INTERNAL` yang bernilai `localhost` dihapus dari environment
produksi sebelum restart aplikasi. Kode ini menahan redirect buruk sebagai
lapisan proteksi, tetapi environment produksi tetap harus bersih.
