# Update 2026-07-22 15:43 Codex

## Scope

Perbaikan arsitektur auth untuk subdomain Altora agar login/register tetap memakai komponen bersama, tetapi logo, warna, judul, dan preset bisnis otomatis mengikuti vertical dari hostname.

## Perubahan

- Menambahkan `src/components/auth/auth-shell.tsx` sebagai shell shared untuk halaman auth.
- `src/app/login/page.tsx` sekarang hanya memanggil `AuthShell` + `LoginForm`.
- `src/app/register/page.tsx` sekarang hanya memanggil `AuthShell` + `RegisterForm`.
- `src/components/register-form.tsx` menerima `lockedBusinessType` dan `lockedBusinessLabel`.
- Register di subdomain vertical tidak lagi menampilkan semua pilihan jenis usaha.
- Register di subdomain vertical melewati tahap kustomisasi modul karena vertical sudah prebuilt.
- Password register sekarang punya tombol tampil/sembunyi, konsisten dengan login.
- `src/app/register/actions.ts` memaksa `businessType` dari hostname vertical agar hidden input tidak bisa di-spoof.
- `src/lib/business-modes.ts` menambahkan mapping vertical ke business mode database yang saat ini valid.

## Catatan Teknis

Database enum `BusinessType` saat ini belum punya tipe granular seperti `PABRIK`, `ECOMMERCE`, `SUPERMARKET`, `TEAMS`, dan `ACCOUNTING`.
Untuk mencegah error register, vertical baru dipetakan sementara ke mode database yang kompatibel:

- Cafe -> `CAFE`
- Toko, E-commerce, Supermarket -> `TOKO`
- Laundry -> `LAUNDRY`
- Counter, Jasa -> `COUNTER`
- Pabrik, Company, Teams, Accounting -> `COMPANY`

Ke depan kalau ingin analitik dan routing lebih spesifik, buat migrasi enum/field baru secara terencana, jangan langsung kirim nilai vertical ke `Tenant.businessType`.
