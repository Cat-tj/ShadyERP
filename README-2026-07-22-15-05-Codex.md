# Update 2026-07-22 15:05 - Codex

## Teams Authentication Routing

### Root Cause

`teams.altora.my.id` sebelumnya diproxy oleh Caddy ke landing app legacy pada
port `3003`. Aplikasi tersebut hanya memiliki halaman marketing sehingga
`/login` dan `/register` mengembalikan HTTP 404.

App modular Altora pada port `3000` sudah dapat melayani kedua route untuk
hostname Teams dan membuat callback cookie ke
`https://teams.altora.my.id`.

### Perubahan

- VPS Caddy telah dialihkan dari `127.0.0.1:3003` ke `127.0.0.1:3000` setelah
  konfigurasi tervalidasi.
- `ops/Caddyfile.altora-main` sekarang mencakup `ecommerce.altora.my.id` dan
  `teams.altora.my.id`.
- Konfigurasi Teams legacy diarahkan ke port `3000` juga untuk mencegah deploy
  berikutnya memulihkan proxy yang rusak.

### Verifikasi Produksi

- `https://teams.altora.my.id/` -> HTTP 200
- `https://teams.altora.my.id/login` -> HTTP 200
- `https://teams.altora.my.id/register` -> HTTP 200
- Caddy tervalidasi dengan `caddy validate` lalu berhasil di-reload.

### Rollback

Backup server dibuat di `/etc/caddy/Caddyfile.backup-20260722-150207`.
Mengembalikan backup itu lalu reload Caddy akan memulihkan konfigurasi lama,
namun login Teams kembali tidak tersedia.
